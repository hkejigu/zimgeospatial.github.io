    Welcome

    Data on Zimbabwe is pretty scattered all over the web and efforts by the national data custodians to implement distribution infrastructure/ initiatives are minimal if existent. One usually finds this data embedded in pdfs, spreadsheets or images. Making any use of the same requires converting to the appropriate format of which time one may not have.Additionally there is lack of contextual exposure of data resulting in stifled innovative use of such. Try finding Murerekwa Primary School for instance on an online map..[now that you're back, from GeoNames, if you tried hard enough), read on!

    Which Zimbabwe?

    In case you're here by accident but would stay, Zimbabwe is a nation in Africa. The map below should (hopefully) help you get oriented.

    Project Motivation

    • Pool together usable geospatial data on Zimbabwe in a somewhat 'quickly' usable format from whichever source inorder to promote innovative use of data.

    Repository Structure

    Each repository contains a set of files which are intended to serve varying purposes. The list below gives an insight into each file's contents and what purpose each file serves.

    • README.md :: Contains the metadata about the repository. To be used for checking on data currency and attribution.
    • filename.shp :: Is a compressed (zipped) version of the data in shapefile format. Due to field_name size restrictions of the shapefile format it is advisable to interpret the field names in the associated README.md file.
    • filename.geojson :: Is the spatial data. To be used for previewing the data and for manipulation with tools such as GeoJson.io - a quick, simple tool for creating, viewing, and sharing maps. And GitSpatial - a spatial API that fronts and syncs your GeoJSON hosted by GitHub.
    • filename.topogeojson:: Is the spatial data in topogeojson format. To be used for data previewing purposes for large datasets.
    • filename.csv ::: Is the spatial data in Comma Separated Values. To be used for searching and having a visual peek at the attribute data.

    Participating

    You're welcome to consume and edit datasets. We're particularly interested in having up-to-date data and correct attributes, spellings e.t.c. Be sure to feedback via email (zimgeospatial [at] gmail.com] or twitter @zimgeospatial

    • 1. If you're geeky, folk, edit, submit pull request.
    • 2. If you've no idea what 1 was about, just download the zipped shapefile and you are good to go.
    • 3. Or simply email the data you wish to share or intend to receive.
